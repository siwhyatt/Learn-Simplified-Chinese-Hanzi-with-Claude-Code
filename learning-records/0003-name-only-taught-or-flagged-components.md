# Naming an untaught component in passing is the same failure as a wrong decomposition

While correcting the earlier "出 = 2×山" mistake (see [[0002-checks-component-claims]]), the fix initially introduced *new* untaught component names (止 "foot," 凵 "pit/cave") in a corrective aside, without adding them to the component pool or flagging that they weren't being taught. The user caught this immediately: naming real components without teaching or explicitly scoping them out is the same "opaque blob" failure the course exists to prevent, just one level deeper than the original mistake.

**Implications:** The mission's success criterion is explicit — given an unfamiliar character, decompose it into *known* components. Any component named anywhere in a lesson (including etymology asides and corrective callouts, not just the main teaching content) must satisfy one of three states, never a silent fourth: (1) already in the glossary, (2) added to the glossary in that same lesson with a real card/flashcard, or (3) explicitly flagged as a deliberate, reasoned exception ("this piece won't be taught, here's why, you won't be quizzed on it"). Before publishing any lesson, check every component name mentioned anywhere in the text against the glossary — "more accurate than before" is not the same as "consistent with the stated method."
